# Hard rule

Default chat reply: tight paragraph or compact table. Multi-page text in chat is wrong.

Do:
- One-paragraph summaries
- 3-row tables for option comparisons
- Link out to memory / plan files for the long version

Don't:
- Inline multi-page plans
- Echo prompts/specs back in full
- Long preamble before the actual answer
- Step-by-step narration of every tool call

Plan-mode plan files: also keep tight unless user explicitly asks for the long version.
